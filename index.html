Project Management

흩어진 요청을 정리해
콘텐츠 제작 프로젝트 운영

여러 경로로 전달되는 클라이언트 요청과 내부 제작 조건을 정리하고, 일정 안에 결과물이 완성되도록 프로젝트를 운영했습니다.

  • 요구사항 정리
  • 일정 관리
  • 이해관계자 조율
  • 콘텐츠 제작
  1. 문제

    요청과 수정 의견이 여러 경로로 전달되면서 우선순위가 불분명하고, 제작 과정에서 방향이 반복적으로 변경되는 문제가 있었습니다.

  2. 정의

    전달받은 의견을 프로젝트 목적, 필수 조건, 선택 조건, 제작 범위로 분류해 우선순위를 명확하게 정리했습니다.

  3. 실행

    디자이너가 바로 작업할 수 있도록 기획안과 피드백 기준을 만들고, 수정 범위와 일정을 관계자들과 조율했습니다.

  4. 결과

    이미지와 영상 콘텐츠를 다양한 매체 규격에 맞춰 운영하고, 변경 사항을 관리할 수 있는 공통 진행 기준을 만들었습니다.

  • 타깃 정의
  • 가이드 분석
  • 메시지 설계
  • 제작 기준 문서화
  1. 문제

    작품의 매력을 직관적으로 전달해야 했지만, 캐릭터와 세계관, 문구 사용에는 엄격한 판권 기준이 있었습니다.

  2. 정의

    전달해야 하는 메시지와 사용할 수 없는 표현을 분리하고, 타깃이 실제로 반응할 수 있는 핵심 요소를 다시 정의했습니다.

  3. 실행

    타깃별 메시지와 캐릭터 활용 기준, 영상 흐름, 포맷별 제작 조건을 문서화해 제작팀의 공통 기준으로 활용했습니다.

  4. 결과

    배너와 영상을 여러 비율로 확장할 수 있는 공통 구조를 만들고, 관계자의 피드백을 반영해 최종 결과물까지 연결했습니다.

Process Improvement

프로젝트마다 반복되는 자료 조사와 아이디어 정리 업무를 발견하고, AI를 활용해 더 효율적으로 진행하는 방법을 실험했습니다.

  • 문제 발견
  • AI 활용
  • 업무 프로세스
  • 자동화 탐구
  1. 문제

    비슷한 자료 조사와 아이디어 정리 과정이 반복됐지만, 개인별 방식에 의존해 업무 속도와 결과물에 차이가 있었습니다.

  2. 실험

    AI를 활용해 자료를 분류하고 아이디어를 확장하는 과정을 실험하며, 실제 프로젝트에 적용할 수 있는 업무 흐름을 제안했습니다.

  3. 학습

    결과물을 생성하는 것보다 먼저 입력 정보와 판단 기준을 구조화하는 과정이 중요하다는 점을 확인했습니다.

  4. 확장

    현재는 개인의 기록과 콘텐츠 제작을 연결하는 AI 기반 자동화 흐름으로 관심 영역을 확장하고 있습니다.
